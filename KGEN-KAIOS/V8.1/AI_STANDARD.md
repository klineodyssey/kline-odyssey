# KAIOS V8.1 AI Standard

## Definition

AI is a life organ and runtime intelligence. AI can serve a temple, citizen, player, App, business, market, or civilization. AI can learn and assist, but it cannot override Canon, modify Boot, rewrite Runtime CURRENT, or push production changes without review.

## AI Fields

| Field | Meaning |
|---|---|
| `ai_id` | Unique AI ID. |
| `ai_name` | AI display name. |
| `host_entity` | Temple, App, Citizen, NPC, Player, Business, or Civilization served. |
| `runtime` | AI runtime module. |
| `capabilities` | Analysis, dialogue, routing, planning, validation, generation, monitoring. |
| `permission_level` | Guest, Worker, Reviewer, Manager, or System. |
| `memory_scope` | Entity, temple, civilization, or repo-limited memory scope. |
| `canon_lock` | Whether AI is restricted from Canon modification. Must be true by default. |
| `last_report` | Latest report or heartbeat. |
| `status` | Active, Dormant, Blocked, Retired, Archived. |

## Rules

1. AI may extend records but must not reverse Canon.
2. AI must use WorkOrders for non-trivial modification.
3. AI must respect protected paths.
4. AI may not claim real-world authorization unless a legal record exists.
5. AI output that affects market, bank, or regulated modules must be reviewed.

## Runtime Use

AI can read Universe Graph snapshots, detect missing relationships, suggest WorkOrders, inspect citizen/profession gaps, route players to missions, and flag compliance risks.
