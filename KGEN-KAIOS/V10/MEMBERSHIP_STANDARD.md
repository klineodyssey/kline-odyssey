# Membership Standard

**Document ID:** KAIOS-V10-MEMBERSHIP-STANDARD  
**Version:** V10.0  
**Status:** Draft for Review

Membership is the role and permission layer for KGEN. It is not KYC, not banking identity and not a legal account system in V10.

## Membership Levels

| Level | Role |
|---|---|
| Guest | Read-only visitor |
| Registered | Basic account holder |
| Verified | Verified prototype identity |
| Temple Owner | Temple manager |
| Land Owner | Land manager |
| Business Owner | Store or service operator |
| Developer | Plugin / App builder |
| Operator | Operational staff |
| Governor | Governance participant |
| Administrator | System administrator |
| Founder | Founding authority |

## Member Fields

Member ID, roles, permissions, verification status, reputation, linked wallet profiles, audit history and risk flags are required fields.

## Boundary

Verified membership in V10 is conceptual. It must not be represented as real-world KYC approval without legal process and regulated provider integration.
