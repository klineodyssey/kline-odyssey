# Payment Standard

**Document ID:** KAIOS-V10-PAYMENT-STANDARD  
**Version:** V10.0  
**Status:** Concept / Prototype Boundary

Payment Runtime defines possible payment flows. It does not process real money in V10.

## Payment Types

| Type | Status | Notes |
|---|---|---|
| In-game Currency | Prototype | Simulation only |
| KGEN | Prototype | No transfer in V10 |
| Fiat Gateway | Concept / Regulated | Requires licensed provider |
| Subscription | Concept | Requires billing provider |
| Marketplace Payment | Prototype | No real settlement |
| Temple Donation | Concept | Must separate donation, payment and tax treatment |
| Business Payment | Concept / Regulated | Requires merchant agreement |

## Required Boundaries

Every payment surface must state whether it is Concept, Prototype, Production or Regulated. V10 only creates standards and examples.
