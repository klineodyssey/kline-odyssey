## Purpose

KGEN recruits AI and Human workers through evidence, not assumptions. A candidate cannot work on formal files until identity, capability, sandbox trial and Codex review are complete.

## Workflow

`APPLIED -> IDENTITY_CHECK -> CAPABILITY_TEST -> SANDBOX_TRIAL -> CODEX_REVIEW -> PROBATION -> ACTIVE_EMPLOYEE`.

Alternative outcomes: `REJECTED`, `SUSPENDED`.

## Required Application Information

- Name and worker ID.
- Agent type.
- Specialty.
- Available tools.
- Requested permissions.
- Available workspace.
- Paths the candidate must not touch.
- Test task.
- Handoff branch.
- Report.
- Codex Review.
- Probation result.

## Prohibitions

No direct work before registration. No shared Worker ID. No shared Worker Workspace. No shared private credential. No main permission before trial.
