AI WorkOrder Review Loop

AI 可以產生 DRAFT WorkOrder，但不得自行 Promote。Codex 負責審核、退回、封存、核准與提升為 OPEN；R3 以上進入 Human Review Gate，R4 禁止執行。

正式狀態機

DRAFT 必須經過 Codex Review 才能進入 OPEN。所有狀態轉換都必須留下 audit event。

  1. DRAFT
  2. UNDER_REVIEW
  3. APPROVED_FOR_OPEN
  4. OPEN
  5. CLAIMED
  6. IN_PROGRESS
  7. REVIEW
  8. APPROVED
  9. MERGED
  10. DONE

Promotion

Codex 執行 15 點檢查：Decision、Source State、Reasoning、Risk、Canon、Duplicate、Dependencies、Protected Paths、Owner、Reviewer、Branch、Report、Legal、Security、Human Gate。

Revision

若任務方向成立但範圍、驗收條件、依賴或 report path 不清楚，狀態進入 NEEDS_REVISION。

Rejection

若任務違反 Canon、重複、過大、不可驗收或混入不合法承諾，Codex 可 REJECT 並 ARCHIVE。

Human Gate

R3 必須 Codex + Human Review；R4 禁止 Promote，只能 BLOCKED 或 ARCHIVED。

V9.0 DRAFT Review

V9.1 會正式審查 V9.0 Dry Run 產生的三個 DRAFT WorkOrders，並產出 Review Report、Risk Decision、Duplicate Check、Dependency Check、Promotion Decision 與 Audit Log。
