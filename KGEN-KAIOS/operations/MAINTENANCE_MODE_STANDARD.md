# KAIOS Maintenance Mode Standard

## Purpose

Maintenance Mode keeps a completed governance recovery safe while an external network dependency is unavailable. It is not permission to repeat handoff cleanup, reopen completed reviews, rebuild Workforce or Decision Engine records, create a V11 proposal, or start V11 development.

## State Model

| State | Meaning | Permitted action |
|---|---|---|
| `NETWORK_WAIT` | GitHub TCP 443, HTTPS, or remote Git access is unavailable. | Record health checks and wait. |
| `NETWORK_RECOVERY` | GitHub connectivity returned. | Fetch once and compare the recorded handoff tips. |
| `CLOSEOUT_READY` | Remote main and all 21 recorded handoff tips are unchanged. | Commit and push declared closeout metadata. |
| `CLOSEOUT_VALIDATING` | Closeout was pushed. | Wait for Pages and validate required health routes. |
| `HUMAN_APPROVAL_WAIT` | Readiness is `READY`; V11 remains unstarted. | Wait for explicit Human approval `START V11`. |
| `BLOCKED` | A tip changed, a gate failed, or evidence is incomplete. | Record evidence and request the required decision. |

## Maintenance Loop

The loop is run every 15 minutes by an external scheduler or when the Human asks for a manual check. This repository does not claim that a background process is running.

Run in this order:

```powershell
Test-NetConnection github.com -Port 443
curl.exe -I https://github.com
git ls-remote origin
Resolve-DnsName github.com
```

Also verify the published GitHub Pages homepage and the most recent Pages deployment. Every result must be appended to `maintenance_log.jsonl`.

## Network Recovery Boundary

When TCP 443, HTTPS, DNS and `git ls-remote` recover, Codex may only:

1. run `git fetch origin --prune` from the clean Codex review worktree;
2. compare the 21 recorded handoff branch tips with `KGEN-AI-Company/reports/V11_READINESS_HANDOFF_RECONCILIATION.md`;
3. stop in `BLOCKED` if any tip is missing or changed;
4. commit the declared closeout metadata if every tip is unchanged;
5. push without force;
6. wait for GitHub Pages;
7. rerun only the V11 readiness gate; and
8. enter `HUMAN_APPROVAL_WAIT` when every gate passes.

This flow must not rerun Recovery, Handoff Cleanup, handoff Review, V11 proposal generation, Workforce construction, or Decision Engine construction.

## Working Tree Rule

`SYSTEM_DIRTY` and `REPORT_GENERATION_DIRTY` are separate conditions. Unrelated or executable changes are `SYSTEM_DIRTY` and block closeout. The declared maintenance, decision, attendance, WorkQueue and readiness records generated by this closeout are `REPORT_GENERATION_DIRTY`; they may be committed together and do not make the readiness gate fail by creating their own report.

Human Main remains isolated. Codex must not reset, stash, clean, delete, merge, or edit the Human workspace.

## Required Evidence

The machine-readable source for current maintenance state is `maintenance_snapshot.json`. Append-only events are stored in `maintenance_log.jsonl`. The public dashboard model is `maintenance_dashboard.json`. Major transitions must also be appended to `KGEN-KAIOS/decision/decision_log.jsonl`.

## Safety Boundary

No maintenance action may modify contracts, wallet Runtime, bridge, Temple 12345 Runtime, Runtime CURRENT, final-whitepaper, KGEN Token contract, secrets, or uncommitted Human files. No force push is permitted.
